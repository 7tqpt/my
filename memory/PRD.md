# PRD — نظام إدارة العقارات (Property Management System)

## Original Problem Statement
المستخدم قدّم نظام Laravel + MySQL "لإدارة العقارات" وطلب إعادة بنائه بالكامل مع الحفاظ على نفس التصميم والوظائف. النظام SaaS لإدارة العقارات: لوحات تحكم، إدارة العقارات/الملاك/المستأجرين، عقود، فواتير المرافق، متابعة مالية، تقارير PDF/Excel، وقيود علائقية صارمة.

## Language & UX
- الواجهة والأخطاء يجب أن تكون بالعربية (RTL) بشكل رئيسي، مع دعم الإنجليزية.

## Tech Stack
- Frontend: React + Tailwind + Recharts
- Backend: FastAPI + Pydantic
- Database: MongoDB via Motor
- Auth: JWT مع جلسة واحدة نشطة + انتهاء صلاحية بعد 3 دقائق خمول

## Implemented Features
- تسجيل دخول / تسجيل خروج مع تنفيذ جلسة واحدة فقط
- CRUD كامل لـ 13 وحدة: الملاك، العقارات، الوحدات، المستأجرين، العقود، المدفوعات، المصروفات، الصيانة، فواتير الكهرباء والمياه، المستخدمين، الإعدادات، لوحة التحكم، التقارير
- تصدير PDF/Excel لكل الجداول + عقد رسمي احترافي
- بحث شامل عبر كل الوحدات
- صفحات معاينة (Preview) لكل كيان مع الجداول المرتبطة به
- قيود حذف علائقية صارمة برسائل عربية (لا يمكن حذف مالك له عقارات، وحدة لها عقود، إلخ)
- تصميم responsive للجوال والحاسوب

## Data Integrity Rules
- لا تُحذف كيان مرتبط ببيانات أخرى (owner → properties, property → units, unit → contracts/bills/maintenance, tenant → contracts)
- لا يُحذف حساب admin الرئيسي أو آخر مدير في النظام
- لا يُكرَّر اسم / رقم جوال / رقم هوية
